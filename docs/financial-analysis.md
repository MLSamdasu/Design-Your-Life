## 2. 수익 모델 및 가격 전략

### 2.1 무료 출시 전략의 타당성 분석

무료 출시 전략은 이 프로젝트에 적합하다. 근거는 다음과 같다.

**타당한 이유:**
1. **시장 진입 장벽 제거**: 생산성 앱 시장은 포화 상태이며, Todoist, TickTick, Google Calendar 등 강력한 경쟁자가 존재한다. 유료로 시작하면 사용자 획득이 극도로 어렵다.
2. **네트워크 효과 부재**: 이 앱은 개인 생산성 도구로, 소셜/협업 기능이 없어 네트워크 효과가 발생하지 않는다. 사용자 수 자체보다 리텐션이 중요하다.
3. **인프라 비용 구조**: Firebase 무료 티어가 10,000 MAU까지 사실상 무료로 운영 가능하므로, 초기 비용 부담 없이 사용자 기반을 확보할 수 있다.
4. **데이터 축적 필요**: 사용자의 습관 데이터, 목표 진행률 등이 쌓여야 앱의 가치가 증가한다. 무료로 시작하여 데이터 lock-in 효과를 만든 후 프리미엄 전환이 효과적이다.
5. **한국 시장 특성**: 한국 사용자는 앱 구매보다 인앱 구독에 더 익숙하다. "무료 다운로드 후 프리미엄 구독" 모델이 한국 시장에서 검증된 전략이다.

**리스크:**
- 수익화 전환 시점을 놓치면 영원히 무료 앱으로 인식될 수 있다.
- 초기 무료 사용자가 유료 전환을 거부할 가능성이 존재한다.

### 2.5 LTV (고객 생애 가치) 추정

#### 2.5.1 핵심 가정

| 지표 | 가정 값 | 근거 |
|---|---|---|
| 무료 -> 유료 전환율 | 3~5% | 생산성 앱 업계 평균 2~7% |
| 월간 이탈률 (Churn Rate) | 8% | 구독 앱 평균 5~12% |
| 평균 구독 기간 | 12.5개월 | 1/이탈률 = 1/0.08 |
| ARPU (유료 사용자) | 3,315원/월 | 수수료 제외 실수령 |

#### 2.5.2 LTV 계산

```
유료 사용자 LTV = ARPU x 평균 구독 기간
                = 3,315원 x 12.5개월
                = 41,438원 (약 41,000원)

전체 사용자 LTV (blended) = 유료 사용자 LTV x 전환율
                          = 41,000원 x 4% (중간값)
                          = 1,640원/사용자
```

| 시나리오 | 전환율 | 이탈률 | 평균 구독 기간 | 유료 LTV | Blended LTV |

### 3.3 재무 리스크 평가

#### 3.3.1 Firebase 비용 폭증 리스크 [High]

| 시나리오 | 발생 가능성 | 예상 영향 | 대응 방안 |
|---|---|---|---|
| 봇/스크래퍼 공격으로 Firestore 대량 읽기 | 중간 | 일일 수백만 reads 발생, 월 수십만원 과금 | Firestore Rules 강화, Firebase App Check 적용, 일일 예산 알림 설정 |
| 비효율적 쿼리로 reads 폭증 | 높음 | 예상의 5~10배 reads 발생 | 쿼리 최적화, 캐시 전략 강화, Firebase Performance Monitoring |
| 무한 루프 버그로 writes 폭증 | 낮음 | 단시간 대량 writes 발생 | 클라이언트 debounce, 일일 쓰기 상한선 설정 |
| Hosting 대역폭 DDoS | 낮음 | 월 수십~수백만원 과금 | Firebase Hosting은 자체 DDoS 보호 제공, Blaze Plan 예산 알림 설정 |

**필수 대응책:**
1. Firebase Console에서 일일 예산 알림을 설정한다 (예: $10/일 초과 시 알림).
2. Blaze Plan 전환 시 예산 상한(Budget Alert)을 반드시 설정한다.
3. Firebase App Check를 활성화하여 비정상 클라이언트의 접근을 차단한다.

#### 3.3.2 무료 사용자만 유입될 리스크 [High]

| 지표 | 위험 신호 | 대응 |
|---|---|---|
| 무료->유료 전환율 | 2% 미만 | 프리미엄 기능 재검토, 가격 조정, 무료 기능 일부 제한 |
| 프리미엄 기능 사용률 | 무료 사용자의 프리미엄 기능 터치율 10% 미만 | 프리미엄 기능 노출 개선, 체험 기간 제공 |

### 4.2 핵심 권고사항

**즉시 실행:**
1. Firebase Blaze Plan으로 전환하되, 일일 예산 알림을 $5로 설정한다.
2. Firebase App Check를 활성화하여 비정상 트래픽을 차단한다.
3. Firestore 쿼리에 반드시 `limit()`과 캐시 전략을 적용한다.

**출시 후 3개월 이내:**
4. 사용자 행동 데이터를 수집하여 실제 Firestore 사용량을 모니터링한다.
5. 실제 DAU/MAU 비율과 세션당 쿼리 수를 측정하여 비용 예측 모델을 보정한다.

**출시 후 6개월 (프리미엄 출시):**
6. 프리미엄 구독을 월 3,900원 / 연 35,900원으로 출시한다.
7. 무료 습관 개수를 5개에서 3개로 제한하여 전환 유도한다.
8. 7일 무료 체험 기간을 제공하여 프리미엄 경험을 유도한다.

**MAU 100,000 도달 시:**
9. Firebase -> 자체 서버 마이그레이션 타당성을 재검토한다.
10. 비용 구조 대비 수익성을 분석하여 마이그레이션 결정을 내린다.

### 4.3 최종 결론

Design Your Life 프로젝트는 재무적으로 매우 안전한 구조를 갖추고 있다. Firebase 서버리스 아키텍처 덕분에 초기 투자가 거의 없고, 사용량에 비례하는 비용 구조는 스타트업 리스크를 최소화한다. 개인 프로젝트로 진행할 경우 직접 비용 5만원 미만으로 시작할 수 있으며, 10,000 MAU까지 월 25,500원 수준의 운영비만 발생한다.

수익화는 출시 6개월 후 프리미엄 구독 모델로 시작하는 것을 권고하며, 기본 시나리오 기준 18개월 시점에서 월 130만원 이상의 MRR을 기대할 수 있다. 치명적 재무 리스크는 발견되지 않았으며, Firebase 비용 폭증 리스크는 예산 알림과 App Check로 충분히 통제 가능하다.
